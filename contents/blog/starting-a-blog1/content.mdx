# 블로그 만들게 된 이유는

  나만의 블로그를 만들고 싶었던 이유가 솔직히 컸다. <br/> 
  블로그를 만들게 되면 커스텀할수도 있고, 추후에는 고도화하여 더 사용성이 좋게 만들고 싶다. <br/>
  또한 블로그를 만들면서 기술적으로 성장하고 싶었기 때문이다. <br/>
  내가 공부한 기록들도 관리를 할수 있고, 흔적을 차곡차곡 쌓고 싶었다. <br/>

## # 장점을 정리하자면 
  ### 커스텀이 가능하다.
  다른 대형 플랫폼의 블로그를 이용하면 쉽고 빠르게 작성할수 있으나, 직접 만들게 되면 내가 원하는 기능을 직접 구현 해볼수도 있다.
  그리고 불편한 부분들을 직접 수정할수 있기 때문이다.

  ### 공부한것을 기록하고 내가 원하는데로 분류를 하여 정리 할수 있다.
  내가 원하는데로 정리할수 있기 때문에 나중에 다시 찾아보기도 편하고,
  부족한 기능이 있을때마다 추가하면서 더 나은 블로그를 만들수 있을것이라 생각한다.

  ### 기술적으로 성장할수 있다.
  블로그를 만들면서 기술적으로 성장할수 있을 것이다. 
  또한 다른 사람들과 소통하면서 예전에는 몰랐던것들을 알수 있을것이라 생각한다.

  ### 오너쉽을 가지고 좀더 나은 블로그를 만들기 위해 애쓸수 있다. (고도화 가능)
  내가 직접 만들어서 운영하는 블로그여서 그런지 좀도 예착이 생기는거 같다.
  그래서 어떻게 하면 더 최적화 하여 좋은 블로그를 만들수 있을지 스스로 고민하고 여러가지 방법을 찾아보며서 성장하는거 같다.
  회사의 서비스를 만들때도 이러한 마음 가짐으로 만들어야겠다고 생각한다.


## # 단점

  ### 시간이 많이 든다.
  블로그를 만들면서 시간이 많이 들어가게 되지만, 하지만 이러한 시간이 투자가 되어 나중에 더 좋은 블로그를 만들수 있을것이라 생각한다.
  그리고 안되는 문제들을 부딪히면서 공부하게 되어 더 많은것을 얻을수 있을것이라 생각한다.
  또한 내가 원하는 기능을 직접 구현해보면서 더 좋은 블로그를 만들수 있다는 점이 재밌을꺼 같다.

  ### 불편할수 있다.
  블로그를 직접 만들면서 어떠한 사이드이펙트가 생길수 있지만, 이러한 점들을 해결하면서 더 좋은 블로그를 만들수 있을것이라 생각한다.




## 개발 환경

  ### Github
    ![Github](./01.png)
    로그를 남겨 어떠한 이슈가 있었는지, 어떠한 기능을 추가했는지 등등을 남기기 위해서 사용합니다.
    깃허브를 저장소 및 백업용으로 두고, vercel에 배포를 하기 위해서 사용합니다. 

  ### Next.js
    ![Next.js](./03.webp)
    Next.js는 React를 기반으로 한 프레임워크입니다. 
    Next.js를 사용하면 SSR을 지원하고, SEO에 좋다는 장점이 있습니다.
    또한 Next.js는 Vercel과 호환성이 좋아서 Vercel에 배포를 하기에 좋습니다. <br/>
    이러한 기능을 사용하여 최적화를 할수 있다. 각각의 페이지에 맞는 메타데이터를 설정하여 SEO를 최적화 할수 있습니다. <br/>
    > ex )`getStaticProps`, `getServerSideProps`, `generateMetaData`, `generateStaticParam` `generateSitemaps` 등등...


    추후에 글을 정리해서 올릴 예정입니다.


  ### Vercel
    ![Vercel](./02.jpeg)
    Vercel은 Next.js를 지원하는 호스팅 서비스입니다. 
    Next의 Docs에서도 Vercel을 추천하는 이유가 있습니다. 
    Vercel은 무료로 사용할수 있고, 빌드시간이 빠르다는 장점이 있습니다.
    또한 Vercel은 Github과 연동이 되어있어서 Github에 있는 저장소와 연동하여 배포를 할수 있습니다.


  ### Tailwind CSS
    ![Tailwind CSS](./04.webp)
    Tailwind CSS는 CSS 프레임워크입니다. 
    Tailwind CSS를 사용하면 CSS를 직접 작성할 필요가 없고, 클래스를 이용하여 디자인을 할수 있습니다.
    또한 Tailwind CSS는 커스텀이 가능하고, 반응형 디자인을 쉽게 구현할수 있습니다.
    그리고 Tailwind CSS는 빌드시간이 빠르다는 장점이 있습니다.
